# VPN 部署手册（Shadowsocks + Clash Verge/Meta）

## 1. 手册目标
本手册用于从零开始重复完成部署工作，产出一个可被 Clash Verge/Meta 使用的稳定节点。

适用范围：
- 服务器系统：Ubuntu 22.04（Debian 系可参考）
- 协议：Shadowsocks
- 客户端：Clash Verge/Meta

最终验收标准：
- 服务器服务状态为 active (running)
- 服务器 443 端口支持 TCP + UDP
- 客户端连通后出口 IP 显示为服务器公网 IP

## 2. 准备清单
部署前确认以下信息：
- 服务器公网 IP
- SSH 用户名与密码（或私钥）
- 云厂商安全组可放行端口（TCP/UDP 443）
- 本地已安装 Clash Verge/Meta

建议约定参数：
- 协议端口：443
- 加密算法：chacha20-ietf-poly1305
- 密码：使用高强度随机串（建议至少 16 位）

## 3. 服务器部署步骤

### 3.1 连接服务器
在本机 PowerShell 执行：

```powershell
ssh -o PreferredAuthentications=password -o PubkeyAuthentication=no root@你的服务器IP
```

如果出现主机指纹变化（REMOTE HOST IDENTIFICATION HAS CHANGED），先执行：

```powershell
ssh-keygen -R 你的服务器IP
```

然后重新 SSH 登录。

### 3.2 更新软件源并安装依赖
登录服务器后执行：

```bash
export DEBIAN_FRONTEND=noninteractive
apt-get update
apt-get install -y shadowsocks-libev jq qrencode
```

### 3.3 写入 Shadowsocks 配置
创建配置文件：

```bash
cat >/etc/shadowsocks-libev/config.json <<'EOF'
{
  "server":"0.0.0.0",
  "server_port":443,
  "password":"请替换为你的强密码",
  "timeout":300,
  "method":"chacha20-ietf-poly1305",
  "mode":"tcp_and_udp",
  "fast_open":false
}
EOF
```

说明：
- 建议使用 server: 0.0.0.0，避免不必要的 IPv6 兼容报错。
- mode 使用 tcp_and_udp，确保 Clash 的 UDP 业务可用。

### 3.4 启动并设置开机自启

```bash
systemctl restart shadowsocks-libev
systemctl enable shadowsocks-libev
```

### 3.5 检查服务状态

```bash
systemctl --no-pager --full status shadowsocks-libev | sed -n '1,30p'
```

期望看到：
- Active: active (running)
- 日志中包含 tcp server listening at 0.0.0.0:443
- 日志中包含 udp server listening at 0.0.0.0:443

### 3.6 检查端口监听

```bash
ss -lunpt | grep ':443' || true
```

### 3.7 放行防火墙与安全组
服务器侧（如启用 UFW）：

```bash
ufw allow 443/tcp
ufw allow 443/udp
ufw reload
ufw status verbose
```

云控制台侧必须同步放行：
- 入站 TCP 443
- 入站 UDP 443

## 4. 生成节点分享信息

### 4.1 生成 SS URI
在服务器执行：

```bash
ENC=$(printf 'chacha20-ietf-poly1305:请替换为你的强密码' | base64 -w0)
SS_URI="ss://${ENC}@你的服务器IP:443#my-ss-443"
echo "$SS_URI"
```

### 4.2 生成二维码（可选）

```bash
qrencode -t ANSIUTF8 "$SS_URI"
```

## 5. 客户端接入步骤（Clash Verge/Meta）

### 5.1 建议使用统一配置文件
你当前工程已提供可用配置模板：
- vpn.yaml

将以下节点参数与实际部署一致：
- server
- port
- cipher
- password
- udp

### 5.2 导入与启用
1. 打开 Clash Verge/Meta。
2. 导入本地配置 vpn.yaml。
3. 启用该配置。
4. 开启 System Proxy。
5. 开启 TUN Mode（推荐）。

### 5.3 策略选择建议
- 节点选择：自动选择 或 手动切换
- 漏网之鱼：节点选择
- 国内直连：DIRECT

## 6. 验收流程
按顺序执行以下检查：

1. 服务器检查
- systemctl status shadowsocks-libev
- ss -lunpt | grep :443

2. 客户端检查
- Clash 日志无持续报错
- 策略组中节点延迟可测

3. 出口检查
- 访问 https://ip.sb 或 https://ipinfo.io
- 期望：出口 IP 为服务器公网 IP

4. 业务检查
- 国外站点（如 YouTube）可访问
- 国内站点访问正常（直连）

## 7. 常见故障排查

### 7.1 SSH 无法登录
- 检查用户名、密码是否正确。
- 强制密码认证后再试：

```powershell
ssh -o PreferredAuthentications=password -o PubkeyAuthentication=no root@你的服务器IP
```

- 若报主机指纹冲突，执行 ssh-keygen -R 你的服务器IP 后重连。

### 7.2 服务启动失败
检查配置 JSON 是否有语法错误：

```bash
cat /etc/shadowsocks-libev/config.json
journalctl -u shadowsocks-libev -n 80 --no-pager
```

### 7.3 客户端可连节点但不能上网
按顺序排查：
1. 云安全组是否放行 TCP/UDP 443
2. Clash 的 System Proxy/TUN 是否开启
3. 策略是否误选 DIRECT
4. DNS 是否被本地软件劫持

### 7.4 规则源下载失败
当前配置使用在线规则源（GitHub）。
可临时切 global 保证先出网，再更新规则；或准备离线规则版本。

## 8. 变更与维护规范
建议团队按以下规范维护：

1. 密码轮换
- 每 30 到 90 天轮换一次。
- 轮换后同步更新 vpn.yaml。

2. 变更记录
- 每次修改记录：时间、修改人、端口、密码版本、验收结果。

3. 备份策略
- 保留最近 2 份可用配置快照。

4. 高可用建议
- 增加备用端口或备用服务器节点。

## 9. 回滚方案
若新配置上线后异常，立即执行：

1. 服务器回滚
- 恢复上一版 /etc/shadowsocks-libev/config.json
- systemctl restart shadowsocks-libev

2. 客户端回滚
- 回退到上一个可用的 vpn.yaml
- Clash 重新加载配置并验证出口

## 10. 一次性执行清单（值班版）
以下清单可直接照着执行：

1. SSH 登录服务器
2. apt-get update
3. 安装 shadowsocks-libev
4. 写入 config.json（443 + chacha20 + 强密码）
5. restart + enable 服务
6. 校验 active + 443 监听
7. 放行安全组 TCP/UDP 443
8. 导出 SS URI
9. 导入 vpn.yaml 到 Clash
10. 开启 System Proxy + TUN
11. 用 ip.sb 验收出口 IP
