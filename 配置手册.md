# Clash Verge/Meta 配置手册

## 1. 目标与现状
本手册用于说明如何使用当前配置文件，让本机通过服务器节点访问外网。

当前配置文件：vpn.yaml
当前节点：
- 类型：Shadowsocks (ss)
- 地址：23.144.132.167
- 端口：443
- 加密：chacha20-ietf-poly1305
- UDP：已开启

## 2. 导入与启用
1. 打开 Clash Verge/Meta。
2. 进入 Profiles（配置）页面。
3. 导入本地配置文件 vpn.yaml。
4. 选择该配置并点击启用。
5. 开启以下开关：
- System Proxy（系统代理）
- TUN Mode（建议开启，兼容更多应用）

说明：
- 当前配置中 mode: rule，表示按规则分流。
- 如果你只想全局走代理，可临时切换模式为 global。

## 3. 配置结构说明
### 3.1 端口
- mixed-port: 7893（推荐客户端统一使用）
- socks-port: 7891
- redir-port: 7892

### 3.2 DNS
- 使用 fake-ip 模式，减少 DNS 污染影响。
- 国内域名优先走国内 DNS，国外域名优先走国外 DoH。
- 若个别网站异常，可加入 fake-ip-filter。

### 3.3 策略组
主要策略组如下：
- 节点选择：总开关（自动/手动/DIRECT）
- 自动选择：url-test 自动探测
- 手动切换：手动指定节点
- AI服务：OpenAI/Anthropic 等流量
- YouTube / Telegram / 国外媒体：按业务分流
- 漏网之鱼：最终兜底

## 4. 推荐使用方式
1. 日常使用：
- 节点选择 = 自动选择
- 漏网之鱼 = 节点选择
2. 国内网站异常时：
- 国内直连 = DIRECT
3. 访问 AI 服务失败时：
- AI服务 = 节点选择 或 手动切换（不要设 DIRECT）

## 5. 验证是否生效
启用配置后，按顺序验证：
1. 打开 https://ip.sb 或 https://ipinfo.io
- 期望结果：出口 IP 显示为服务器 IP（23.144.132.167）
2. 打开 https://www.youtube.com
- 期望结果：可正常访问
3. 打开国内网站（如百度）
- 期望结果：直连且速度正常

## 6. 常见问题与处理
### 6.1 能连节点但无法上网
排查顺序：
1. Clash 是否已启用该配置。
2. System Proxy 与 TUN 是否开启。
3. 节点选择是否误设为 DIRECT。
4. 本地防火墙或安全软件是否拦截。

### 6.2 规则下载失败
你当前配置使用在线 rule-providers（GitHub）。
如果提示 ruleset 下载失败：
1. 先在策略中临时改为 global 确保能出网。
2. 再回到 rule 模式更新规则。
3. 或改成离线规则版本（可单独生成一份）。

### 6.3 某些应用不走代理
1. 确认 TUN 开启。
2. 在 Clash 设置中允许接管更多流量（按客户端版本名称可能不同）。
3. 必要时给该域名/进程追加规则。

### 6.4 节点突然不可用
1. 检查服务器是否在线。
2. 检查云厂商安全组是否放行 TCP/UDP 443。
3. 服务器执行：
- systemctl status shadowsocks-libev
- ss -lunpt | grep :443

## 7. 节点信息修改指引
如需更换服务器、端口或密码，编辑 vpn.yaml 中 proxies 对应条目：
- server
- port
- password
- cipher

修改后：
1. 保存文件
2. 在 Clash 里重新加载配置
3. 重新进行第 5 节验证

## 8. 安全与维护建议
1. 建议定期更换节点密码。
2. 建议新增一个备用端口节点，避免单点故障。
3. 若长期使用，建议升级为更抗干扰方案（如 Reality）。
4. 建议保留一份可用的离线配置，防止规则源临时不可达。

## 9. 快速回滚
如果新改动导致不可用：
1. 将策略切回 global
2. 节点选择改为手动切换并指定可用节点
3. 若仍失败，回退到上一个可用配置文件
